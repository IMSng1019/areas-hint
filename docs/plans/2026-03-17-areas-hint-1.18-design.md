# areas-hint 1.18 Compatibility Design

**Target:** Port the current Fabric mod codebase to Minecraft 1.18 with minimal behavioral change.

**Scope:**
- Update dependency and metadata versions for Minecraft 1.18.
- Repair compile failures one at a time.
- Prefer compatibility-only changes with no feature changes.
- If failures are numerous, adapt by module in this order: networking, rendering, mixin, commands.

**Constraints:**
- Keep existing behavior unchanged.
- Do not suppress errors by commenting out code.
- Prefer minimal API migration over refactors.
- Verify with fresh `./gradlew build` runs after each module stage.

**Execution Notes:**
- Use Context7 first for Fabric API migration references.
- Maintain a running to-do list of compile failures and append new items as they appear.
- Prioritize Mixin injection-point issues before broader API breakage when selecting the next fix.
